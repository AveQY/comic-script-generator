# 编辑与续写工作流

当用户要求修改现有分镜或续写时，遵循以下流程。

## 读取上下文

### 读取摘要优先

始终先读取 `summary.md`，了解整体进度，再读取具体集数：

```
1. summary.md — 项目全局状态
2. characters.md — 当前角色档案
3. foreshadowing.md — 未回收伏笔
4. episodes/epXXX_*.md — 具体集数（按需读取）
```

### 读取上一集末段

续写时，必须读取上一集的**最后 3-5 个场景**，确保情节衔接自然：

```
读取 ep002_回忆往昔.md 的最后 3000 字符
```

## 局部修改流程

### 用户指定场景修改

当用户说"把第3集的第7个场景改一下"：

1. 定位到具体场景：`ep003_xxx.md` 中的 `## Scene 7`
2. 只修改该场景的内容，不动其他场景
3. 修改后检查：
   - 修改后的对话是否与前后场景连贯？
   - 角色语气是否符合性格？
   - 是否引入了新的伏笔（需要记录）？

### 用户指定情节修改

当用户说"让阿明在第5集发现真相"：

1. 检查 foreshadowing.md，看是否有相关伏笔
2. 如果伏笔已埋设，确认回收位置合理
3. 如果伏笔未埋设，在修改处新增伏笔记录
4. 更新 summary.md 中对应集数的"关键转折"

## 伏笔管理

### 新增伏笔

在 `foreshadowing.md` 的"未回收"部分添加：

```markdown
- [ ] 阿明发现父亲旧照片中的神秘女子（第5集埋下）
```

### 回收伏笔

1. 从"未回收"移动到"已回收"
2. 标注回收位置：

```markdown
- [x] 阿明父亲的遗物（第2集埋下，第8集回收）
```

### 伏笔冲突检查

续写前检查 foreshadowing.md：
- 是否有即将到期的伏笔（用户期望回收但还没回收）？
- 是否有重复的伏笔（同一件事埋了两次）？

## 角色一致性检查

续写时检查 `characters.md`：
- 新场景中角色的对话是否符合已记录的性格？
- 是否需要更新角色档案（新增关键情节）？

## 写作风格延续

- 保持与已写集数相同的对话风格
- 如果用户之前指定了 art_style，新场景也要符合
- 分镜密度模式不能中途改变（同一项目必须一致）

## 中断恢复

如果生成过程中断（上下文不够或用户手动停止）：
1. 读取已完成部分的最后场景
2. 根据 summary.md 中的摘要继续
3. 不要重新生成已完成的场景
